# 将包的 SPEC 从 vllm-specs 仓库迁移到 openRuyi 仓库

以迁移 `foo` 包为例。以下步骤中，**"分支是新建的"** 指本次任务在步骤 2 中创建了该分支；**"分支已存在"** 指步骤 2 中该分支已存在并执行了 rebase。

## 步骤

### 1. 同步 openRuyi 仓库的 main 分支

进入 openRuyi 仓库，切换到 `main` 分支，拉取最新代码。

### 2. 准备工作分支 `new-foo`

- 若 `new-foo` 分支**不存在**：创建该分支。
- 若 `new-foo` 分支**已存在**：将其 rebase 到 `main` 分支上。

### 3. 切换到 `new-foo` 分支

### 4. 复制 SPEC 文件

将 `/vllm-specs/SPECS/foo` 文件夹复制到 openRuyi 仓库的 `SPECS/` 目录下。

### 5. 暂存并提交

暂存所有变更，提交信息为 `SPECS: add foo`。

- 若**分支是新建的**：直接提交。
- 若**分支已存在**：使用 `--amend` 修改上一条提交（保持远端只有一个提交）。

### 6. 推送代码

- 若**分支是新建的**：使用 `--set-upstream` 推送。
- 若**分支已存在**：使用 `--force` 强制推送。
