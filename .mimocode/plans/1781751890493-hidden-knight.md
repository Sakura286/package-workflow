# Plan: Review and fix AGENTS.md vs Skill consistency

## Analysis

I've compared `AGENTS.md` (197 lines) with the skill's `SKILL.md` and all 5 sub-files
(workflows/upgrade-package.md, workflows/new-package.md, workflows/fix-build.md,
reference/declarative-build.md, reference/obs.md).

### Issues found

#### 1. Missing `SKIP_TRIGGER_CHECK=1` after `osc ci` (AGENTS.md:121-129)

The "Creating a new OBS package" section says to run `osc ci` but doesn't mention
that `osc ci` itself runs the service. The watcher needs `SKIP_TRIGGER_CHECK=1` in
this case because the spec commit was already on `main` before the OBS package existed.
This is documented in `reference/obs.md:144-146` but absent from AGENTS.md.

**Fix**: Add a note after the `osc ci` command about arming the watcher with
`SKIP_TRIGGER_CHECK=1`.

#### 2. Incomplete watcher event list (AGENTS.md:108)

AGENTS.md lists: `TRIGGERED`, `RESULT`, `DONE`.
`reference/obs.md:93-100` also lists: `TRIGGER-TIMEOUT` and `WARN`.

**Fix**: Add the missing events to AGENTS.md's watcher section, matching obs.md.

#### 3. Minor: no `_service` template (AGENTS.md:126-129)

AGENTS.md tells you to `cp python-torch/_service <pkg>/_service` and edit it, but
doesn't show what the template looks like. The full template is in `reference/obs.md:150-161`.
This is acceptable — AGENTS.md intentionally points to the skill for details.
**No change needed.**

#### 4. Redundancy is by design

AGENTS.md duplicates much of SKILL.md. This is intentional — AGENTS.md is always
loaded as system context, while the skill is loaded on-demand for packaging tasks.
The overlap ensures the agent has baseline knowledge even without loading the skill.
**No change needed.**
