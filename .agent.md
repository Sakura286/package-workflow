# OBS RPM Build Automation - Agent Instructions

## Overview
You are an AI agent helping with RPM packaging for Open Build Service (OBS). 
You have access to automation scripts that can trigger builds, check status, 
fetch logs, and apply fixes to spec files.

## Available Scripts
All scripts are in `scripts/` relative to the workspace root.

| Script | Purpose |
|--------|---------|
| `scripts/obs-trigger-build` | Trigger a rebuild in OBS |
| `scripts/obs-check-status` | Check current build state |
| `scripts/obs-fetch-log` | Download build log |
| `scripts/obs-auto-build` | Full trigger+wait+fetch cycle |
| `scripts/obs-ai-loop` | Full automated build-fix loop |

## Workflow for Build-Fix Loop

When the user asks to "fix the build" for a package, follow this loop:

### Step 1: Trigger Build
```bash
./scripts/obs-trigger-build home:Sakura286:ROCm <package> amd64_build x86_64
```

### Step 2: Wait & Check Status
Poll every 60 seconds until the build reaches a terminal state:
```bash
./scripts/obs-check-status home:Sakura286:ROCm <package> amd64_build x86_64
```
Terminal states: `succeeded`, `failed`, `unresolvable`, `broken`, `blocked`, `excluded`

### Step 3: If Failed - Fetch Log
```bash
./scripts/obs-fetch-log home:Sakura286:ROCm <package> amd64_build x86_64
```
This saves the log to `log/<package>-<timestamp>.log`

### Step 4: Analyze the Error
Read the log file (focus on last 200 lines and grep for errors).
Cross-reference with:
- The spec file at `rocm-specs/SPECS/<package>/<package>.spec`
- Source code in `orig_code/` if needed
- Previous logs in `log/` for patterns

### Step 5: Apply Fix
Edit the spec file using the appropriate tool.
Common fixes:
- Missing BuildRequires
- Incorrect paths or flags
- Compiler/linker errors
- ROCm/HIP configuration issues
- Patch needs updating

### Step 6: Commit & Push
```bash
cd rocm-specs && git add -A && git commit -m "<package>: fix <issue>" && git push
```

### Step 7: Loop
Go back to Step 1 until build succeeds or max 5 iterations.

## Important Notes
- OBS API URL: https://pickaxe.oerv.ac.cn
- The `_service` file uses `obs_scm` to auto-pull from git, so pushing to the specs repo triggers a rebuild automatically (no need to manually run `osc service rr` if the git hook is configured)
- Build repo: `amd64_build`, Arch: `x86_64`
- ROCm builds use clang, not gcc
- Memory-intensive builds (like pytorch) may need special handling
